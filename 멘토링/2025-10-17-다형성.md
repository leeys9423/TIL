# 다형성(Polymorphism)의 정의

## 내가 생각한 정의
**다형성이란 클라이언트에게 구현체 대신 인터페이스를 제공함으로써, 구현체가 변경되어도 클라이언트의 코드는 영향을 받지 않도록 하는 객체지향의 특징이다.**

---

## 왜 이게 중요한가?

구현체가 변경되어도 클라이언트 코드가 변하지 않는다!!

1. **유지보수가 쉬워진다** - 기존 코드를 수정할 필요가 없다
2. **확장성이 좋아진다** - 새로운 구현체를 추가하기만 하면 된다
3. **의존성 관리가 명확해진다** - 추상화(인터페이스)에만 의존한다

---

## 실제 예시

```java
// 인터페이스에 의존
PaymentService paymentService;

// 구현체가 바뀌어도
paymentService = new CreditCardPaymentService();  // 신용카드
paymentService = new KakaoPayPaymentService();    // 카카오페이

// 클라이언트의 코드는 동일하게 동작
paymentService.pay(100);
```

클라이언트(결제 요청자)는 "결제"라는 행위만 신경 쓰면 되고, 어떤 플랫폼으로 결제되는지는 신경 쓸 필요가 없다.

---

## 핵심 포인트

다형성 = **인터페이스를 통한 추상화로 구현의 변화로부터 클라이언트를 보호하는 것**
